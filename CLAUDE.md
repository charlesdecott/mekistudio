# mekistudio

AI dev studio en **pur Python, sans Docker, auto-hébergé** : un seul repo (`C:\mekistudio`, branche `main`) qui se met à jour lui-même. **Vision** : importer mekistudio dans lui-même et le laisser s'améliorer. On reconstruit **petit à petit**, en réimplémentant les concepts des anciennes versions — sans reprendre leur code.

## Architecture
- `mekistudio/backend/` n'importe **jamais** `frontend/` ; `mekistudio/cli.py` est le seul câblage.
- Stack : uv · Typer · FastAPI/uvicorn · Jinja2 · Alpine.js · Pydantic v2 · Claude Agent SDK · pytest.
- CLI : `serve` (canvas sur :8777, bootstrap `.mekistudio/`) · `update` (git pull, code live) · `update --restart` (stop + pull + relance).
- Install global **editable** (`uv tool install --editable .`) → le code est lu en direct depuis le repo.
- Composants : `backend/components/` (primitives Pydantic, union discriminée sur `type`) assemblés en nodes dans `backend/nodes/` (`registry.py` → `NODE_BUILDERS`/`default_canvas`). `CanvasState.nodes` typé `list[Node]`. Topologie built-in : **`kernel → git → { chat, subcanvas → explorateur }`** (brique H). **Node `subcanvas`** : cadre réductible générique qui confine le monde de l'explorateur (explorateur + dossiers + éditeurs) ; ses bornes sont **dérivées** de la bounding-box de son sous-arbre (+ padding + bande titre 26 px) — descendants **exclus** de la collision principale (dérivés de l'arbre `source_id` via `_containedIds` ; le cadre participe comme une seule boîte) ; réduction → tuile (descendants masqués, câbles internes cachés) ; migration auto (`_ensure_builtin_nodes` + `reconcile_source_links`). Nodes dynamiques : `fileeditor`, `folder` (mini-explorateur enraciné, dossiers-en-nodes).
- Câbles/wires **dérivés** d'un parent par node (`Node.source_id`). Parentage **par kind** (`CANONICAL_PARENT_KIND`) pour les built-in, **path-aware** (plus-long-préfixe de chemin, `nodes/parenting.py` pur) pour `folder`/`fileeditor` ; `reconcile_source_links` applique les deux au boot (+ migration des built-in de mauvais kind). Pas d'`edges` persistés (`CanvasState.edges` réservé/inutilisé). Géométries/logiques **pures** côté front (testées `node --test`) : `cables.js` (`MekiCables` — subway 45°, ruban, contournement, `pathBetween`), `collision.js` (`MekiCollision` — collision douce), `folders.js` (`MekiFolders` — chaîne de dossiers complète/compacte), `git-node.js` (`MekiGitNode` — rendu état git), `zonelayout.js` (`MekiZoneLayout` — `radialLayout` arbre radial, `packAround` anneau de fichiers, `freestAngle`, `solve`/`packOutward` réserve), `territories.js` (`MekiTerritories` — hull/blob de zone + `separatePolys` dé-collision par polygones MTV), `subcanvas.js` (`MekiSubcanvas` — `descendants`/`derivedBounds` bornes dérivées du sous-arbre). `canvas.js` les câble en DOM impératif. Node git rafraîchie à la fin de tour (`GET /api/git/branch`, lecture seule). **Placement par ARBRE RADIAL (node-zones)** : chaque dossier est une *node-zone* (tuile 📁 au CENTRE, fichiers directs rangés en ANNEAU autour — `folderBlobCorners` inclut la tuile). `relayoutZones` (re)dispose tout à chaque changement : (1) `_reconcileFileParents` rattache chaque fichier à son dossier le plus profond, MÊME règle que le serveur (`reconcile_source_links`) → fichiers dans la BONNE zone + cohérence live/reload ; (2) `radialLayout` pose l'explorateur (racine épinglée, ancrée sur son HAUT stable) au centre et fait croître les dossiers en RAYONS vers l'extérieur (chaîne droite, fourche en cône) — enfants triés par CHEMIN (reproductible quels que soient les ids) ; (3) `packAround` range les fichiers en anneau (taille de fichier CANONIQUE → indépendant de la mesure live instable) ; (4) `separatePolys` (MTV) garantit un VIDE entre les blobs DESSINÉS. **Déterministe → 0 chevauchement + stable au reload (aucune dérive)** ; mouvement animé (transition CSS), deadband anti-jitter. `fitView` auto-zoome. Câbles backbone (bleu/ambre) **contournent les zones tierces** + mis en avant (CSS) ; câbles fichiers (vert) courts dans leur zone. `editorSpawnPos` ne sert qu'à l'init d'un nouveau node (relayoutZones replace ensuite).
